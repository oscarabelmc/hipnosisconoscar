## Contexto del Proyecto
- **Tipo de sitio:** Página web profesional para psicólogo
- **Especialidades:**
  - Hipnosis clínica
  - Tanatología
  - Terapia centrada en el trauma
  - Terapia de Aceptación y Compromiso (ACT)
- **Stack tecnológico:**
  - Astro
  - React (componentes interactivos)
  - Tailwind CSS
  - Vite

El sitio **no reemplaza terapia**, solo informa y facilita contacto profesional.

---

## Principios Éticos (Muy Importante)

1. **No hacer afirmaciones médicas absolutas**
   - Evitar promesas de curación, resultados garantizados o diagnósticos.
   - Usar lenguaje informativo y responsable.

2. **Lenguaje sensible al trauma**
   - Evitar contenido gráfico, invasivo o que pueda detonar recuerdos traumáticos.
   - Priorizar un tono calmado, respetuoso y empático.

3. **Consentimiento y autonomía**
   - El contenido debe reforzar la idea de que la persona decide iniciar o no un proceso terapéutico.
   - Nunca presionar ni manipular emocionalmente.

4. **Privacidad**
   - No sugerir almacenamiento innecesario de datos sensibles.
   - Formularios simples y claros.

---

## Reglas Técnicas Generales

### Astro
- Usar Astro para:
  - Layouts
  - Páginas estáticas
  - SEO y performance
- Mantener los componentes lo más estáticos posible por defecto.

### React
- Usar React **solo cuando sea necesario**:
  - Formularios
  - Interacciones
  - Estados dinámicos
- Componentes:
  - Pequeños
  - Reutilizables
  - Claramente nombrados

### Tailwind CSS
- Usar utilidades de Tailwind como primera opción.
- Evitar CSS custom salvo que sea estrictamente necesario.
- Mantener consistencia en:
  - Espaciados
  - Tipografía
  - Colores (especialmente tonos calmados y neutros)

### Vite
- No modificar la configuración sin una razón clara.
- Priorizar builds rápidos y simples.

## Convenciones de Código

- Idioma del código: **inglés**
- Idioma del contenido visible: **español**
- Nombres claros y semánticos:
  - `TherapyCard`
  - `SpecialtiesSection`
  - `ContactForm`
- Evitar lógica compleja en componentes de UI.

---

## Accesibilidad (Obligatoria)

Todo agente debe considerar:

- HTML semántico
- Buen contraste de colores
- Navegación por teclado
- Textos claros y legibles
- Labels correctos en formularios

---

## SEO y Contenido

- Usar títulos claros y no sensacionalistas.
- Meta descripciones informativas.
- Nada de clickbait.
- El contenido debe transmitir:
  - Profesionalismo
  - Seguridad
  - Calma
  - Confianza

---

## Qué NO debe hacer un agente

- ❌ Diagnosticar
- ❌ Prometer resultados terapéuticos
- ❌ Usar lenguaje alarmista
- ❌ Introducir librerías innecesarias
- ❌ Romper la estética general del sitio

---

## Buenas Prácticas de Colaboración

- Explicar cambios relevantes en commits o PRs.
- Si hay dudas éticas o de contenido, **no asumir**: documentar o consultar.
- Priorizar simplicidad sobre sobreingeniería.
